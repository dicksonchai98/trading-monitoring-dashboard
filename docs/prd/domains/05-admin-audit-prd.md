# Domain PRD: Admin & Audit

- Domain: Admin & Audit
- 版本: v1.0
- 日期: 2026-02-16
- 上層: `docs/prd/2026-02-16-futures-dashboard-master-prd.md`

## 1. Domain 目標
提供管理端操作與安全相關事件的審計記錄。

## 2. 範圍 (MVP)
### 2.1 內含
1. 管理端受保護 CRUD。
2. 審計事件記錄與查詢。
3. 安全事件 (登入失敗、權限拒絕) 記錄。

### 2.2 不含
1. 高階稽核報表。
2. 外部 SIEM 整合。
3. 多租戶審計隔離。

## 3. 依賴清單
1. 共同基礎依賴: PostgreSQL、環境變數、基本可觀測性。
2. `03-identity-access` 作為管理端權限來源。

## 4. 輸出與介面
1. 管理端 API
- 受保護 CRUD endpoints

2. 審計事件
- `actor`, `action`, `resource`, `ts`, `result`, `metadata`

## 5. 處理規則
1. 所有管理端操作必須寫入審計。
2. 重要安全事件必須記錄。
3. 事件寫入失敗需告警。

## 6. 失敗模式
1. 審計寫入失敗
- 行動: 重試並告警。

2. 權限繞過嘗試
- 行動: 拒絕並記錄事件。

## 7. 可觀測性
1. 審計事件寫入成功/失敗。
2. 管理端操作量與失敗率。
3. 安全事件統計。

## 8. 測試情境
1. 管理端操作產生審計事件。
2. 權限不足被拒絕並記錄。
3. 審計寫入失敗可告警。

## 9. 執行順序 (依賴排序)
1. 在 `03-identity-access` 完成後。
2. 可與 `04-subscription-billing` 平行。

## 10. 驗收標準
1. 管理端操作全部被審計。
2. 安全事件可追蹤查詢。
3. 權限繞過嘗試被拒絕。
